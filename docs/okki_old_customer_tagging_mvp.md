# OKKI Old Customer Tagging MVP

## MVP 目标
- 先把 OKKI 老客户完成可审计、可回读验证的等级/标签打标流程。

## 输入
- 未打标老客户列表（客户 ID/名称/URL）。

## 读取范围
- 动态
- 资料
- 公司常用信息
- 公司其他信息

## 第一版只写字段
- 客户等级
- 客户标签

## 暂不自动写字段
- 网站
- 国家
- 邮箱
- 联系人职位
- 公司规模

## 打标流程
1. 读取（客户详情 + 资料字段）
2. 分析（规则/模型生成建议）
3. dry-run（只输出建议，不写入）
4. 写入计划（目标字段、目标值、风险检查）
5. 单客户确认
6. 写入执行
7. 回读验证

## 失败处理
- 字段缺失：返回 null 并记录 `missing_fields`
- 页面模式不一致：先 `detect_page_mode()` 再分支处理
- 标签已存在：跳过重复标签，只保留缺失标签
- 客户名不匹配：停止写入并记录异常
- 写入后回读失败：标记失败并进入人工复核

## 安全原则
- 默认 `dry_run=True`
- 禁止固定 `@eXX` 和坐标
- 语义锚点定位（字段名/分组名/tab 名）
- 每次 tab 切换/分组展开后重新 snapshot

## 本轮声明
- 本轮仅做资料字段全量侦查与代码骨架更新。
- 本轮没有修改 OKKI 数据。
